### 1. **Ringkasan & Tujuan Project**

TrafficBuster adalah aplikasi simulasi trafik website, berbasis Windows WPF .NET dan backend Node.js Playwright, yang secara modular dapat menggandakan ribuan simulasi kunjungan, perilaku manusia, tes performa, audit SEO, UX, keamanan, proxy rotation, serta export data analisa siap AI dan pengujian produksi. Cocok untuk dev, devops, dan QA.[1][2][3]

***

### 2. **Keunggulan TrafficBuster**

- Simulasi manusia (scroll, klik, dwell, bounce, interaction, multi-platform, fingerprint unik).
- Import URL massal semua format (.txt, .csv, .xlsx, .docx).
- Proxy list rotasi, tes kecepatan, geo-detect, DNS custom.
- Platform profile OS/Device/Browser kombinasi random/realistis.
- Modular, per-tab, concurrency, worker pool, session, task scheduler.
- Export log CSV, JSON, audit otomatis, Safe Mode.
- Backend Node.js Playwright, scaling REST-WebSocket, remote server, API key, TLS, audit trail.
- Configurable JSON/YAML, SQLite untuk history dan log.
- Checklist AI coder: semua bab, API, pola, dan DTO tersedia.

### 5. **Perincian Bab & Subbab Setiap Tab**

#### **Tab General [Target URL & Traffic Source]**
- Import, edit, fetch file (.txt, .csv, .docx, .xlsx)
- Validasi regex
- Setting mass traffic type: Direct, Organic, Referral, Random
- Organic Traffic: Search Engine & Keyword Table
- Referral: Source dropdown + Custom
- Table: URL, type, source, keywords, edit inline

#### **Tab GeoLocation [Proxy, DNS, Country]**
- Input proxy (host:port:username:password)
- Import bulk, format smart
- Table: Proxy list, credentials, test sekali, country by geoip, status, type
- DNS setting: Proxy DNS, Host DNS, Custom DNS
- Global Proxy Enable/Disable switch
- Status: kecepatan, failover, Test proxy sekali

#### **Tab Platform [Device, Browser, Fingerprinting]**
- List/Template: OSDevice, Version, Browser, Resolution
- Droplist & Randomizer
- Otomatis UA builder
- Tabel: PlatformProfile

#### **Tab Settings [Runtime, Policy, Backend API]**
- Instance count, concurrency, delay, rotation, session, time-based
- Proxy switching policy: PerFlow, OnceAllPages, TimeBased ms/random
- Platform switching: single/random/per-page
- Safe Mode on/off, click policy ratio
- Backend config: Node API host, port, TLS, API Key, autostart, remote, SSH
- Preset config: Max Session, Max Bounce, Default, Custom

#### **Tab Run/Monitoring [Execution, Log, Export]**
- Start, dry-run/full, pause/resume/stop
- Live log: progress bar, status per flow, per-instance detail
- Export: CSV, JSON, PDF, auto save
- Metrics: CPU, RAM usage, effective flows completed
- Logging: Audit trail, error handle, consent file, report generation

### 7. **Workflow Detail & Non-Overlapping Execution**

- Tidak ada tumpang tindih antar tab/fungsi. Setiap import/setting/tes proxy hanya tercatat sekali, tidak mengulang cycling/proxy sebelum run engine.
- Worker Pool, task scheduler, concurrency InstanceCount, semaphores, atomic data save.
- Implement multi-instance, failover, retry/backoff, autoscale throttling.
- Export log dan config selalu via API, tidak lokal-stateless.

***

### 8. **API REST/WebSocket Backend Node.js**

- REST: /api/v1/project/load, /run/start, /run/stop, /run/status
- WebSocket streaming: log, status update
- Secure API Key, TLS setup, rate limit
- Validasi JSON payload, consent logic, endpoint segregasi dry-run vs real-run

### 11. **Edge Case & Robustness**

- Proxy failure: hot-swap, retry/backoff/jitter, exponential backoff on site-blocking
- CPU/memory: autoscale concurrency, reduce parallelism otomatis
- Logging: audit, obfuscated data
- Data corrupt recovery: atomic save/backup temp, restore safe
- Consent & SafeMode: klik eksternal require signed consent & log

***

### 12. **Testing, Metrics, Performance**
- Acceptance criteria dan unit test plan lengkap[2][3]
- Integration & performance testing 10k–100k flows, concurrency 10–50 instances
- Metrics: RPS, latency, fail/success, error rate per source/proxy/target, click distribution, memory footprint

***

### 13. **Security, Consent, Audit**
- Proxy encrypted, consent timestamp, API key usage, TLS optional
- Audit log click, flow, error, export log with PII masking
- SafeMode: external ad click di-block default, unlock by consent
