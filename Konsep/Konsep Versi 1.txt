**Traffic Buster** adalah aplikasi berbasis **Windows GUI** yang berfungsi sebagai **alat simulasi trafik dan pengujian performa website** secara realistis.
Aplikasi ini dirancang untuk meniru perilaku pengunjung manusia saat menjelajahi situs web â€” termasuk mengakses halaman, berinteraksi, menggulir, dan berpindah antar halaman â€” dalam kondisi dan parameter yang dapat dikustomisasi penuh.

Tujuan utama **Traffic Buster** adalah memberikan **simulasi beban dan perilaku trafik** yang akurat terhadap website milik pengguna sendiri atau proyek web internal, sehingga developer dapat:

* Menguji **stabilitas dan skalabilitas website**,
* Mengamati **respons server terhadap berbagai pola trafik**,
* Dan **memvalidasi kinerja sistem caching, SEO, maupun UX**.

Aplikasi ini mengembangkan konsep dari *Tornado Traffic Generator*, dengan penambahan:

* **Antarmuka modern & user-friendly**,
* **Manajemen proxy yang komprehensif**,
* **Sistem multi-instance paralel**,
* **Simulasi perilaku manusia**,
* **Konfigurasi fleksibel yang bisa disimpan & dimuat ulang**,
* Serta **arsitektur modular berbasis pattern modern (MVC/MVVM)**.

## ðŸ§­ **1. Tab: General â€” Target URL & Traffic Source**

Tab ini merupakan pusat pengaturan utama untuk daftar website atau halaman yang akan diuji.
Di sini pengguna dapat menambahkan, mengelola, atau mengimpor banyak URL sekaligus.

### ðŸ”¹ **Komponen Utama**

#### **A. Input Manual**

* **URL (Wajib)**
* **Traffic Type**:

  * `Direct Traffic`
  * `Organic Traffic`
* **Jika memilih Organic**, maka muncul dua opsi tambahan:

  * **Search Engine Source**

    * Dropdown: `Google, Bing, Yahoo, Baidu, DuckDuckGo, Yandex, Ask, Naver`
    * Field **Keyword** (comma separated)
  * **Referral Source**

    * Dropdown: `Facebook, X (Twitter), Instagram, YouTube, TikTok, Pinterest, LinkedIn, Reddit, Custom`
    * Jika memilih *Custom*, muncul field tambahan untuk mengisi URL referer.

#### **B. Fetch URL (Import URL List)**

Fitur **Fetch URL** kini terintegrasi langsung di tab **General**.
Fungsi ini memungkinkan pengguna untuk mengimpor banyak URL dalam satu waktu dari file eksternal:

* Mendukung format: `.txt`, `.csv`, `.doc`, `.docx`, `.xls`, `.xlsx`
* Sistem otomatis memvalidasi format URL (harus `http://` atau `https://`)
* Setelah file dimuat, pengguna dapat:

  * Mengatur **Traffic Type secara massal** (Direct, Organic, Referral)
  * Atau memilih opsi **Random Traffic Type**
  * Jika memilih *Random Organic Traffic*, maka sistem membagi secara acak tipe dan sumber trafik, tanpa memerlukan keyword.

#### **C. Daftar URL (List Table)**

| No | URL | Website | Traffic Type | Source | Keyword |
| -- | --- | ------- | ------------ | ------ | ------- |

Contoh:

* Jika 1 URL dipilih sebagai *Organic - Google* dengan 5 keyword, maka muncul 5 entri berbeda di tabel.
* Jika diimpor 100 URL dengan opsi *Random Traffic Type*, sistem otomatis membuat variasi sesuai proporsi acak.

## ðŸŒ **2. Tab: Geo Location â€” Proxy Management**

Tab ini digunakan untuk **mengelola dan menguji daftar proxy** yang akan digunakan oleh Traffic Buster.
Sistem mendukung **rotating proxy**, **deteksi lokasi otomatis**, serta **DNS konfigurasi kustom**.

### ðŸ”¹ **Komponen Utama**

* **Proxy Source:** Input manual atau import dari URL/file teks
  Format dukungan:

  * `hostname:port`
  * `hostname:port:username:password`

* **Test Proxy:**
  Melakukan pengujian kecepatan dan validasi konektivitas untuk seluruh daftar proxy.
  Hasil uji hanya dilakukan **sekali** saat proxy ditambahkan.

* **Proxy Table:**
  | No | Proxy | Credential | Speed | Type | Country | Status |

  * **Country**: otomatis ditentukan berdasarkan IP proxy.
  * **Status**: menunjukkan apakah proxy aktif atau gagal terhubung.

* **DNS Configuration:**

  * `Use Proxy DNS`
  * `Use Host PC DNS`
  * `Custom DNS` â†’ bisa diisi manual (misal `8.8.8.8; 1.1.1.1`)

* **Enable / Disable Proxy (Global Switch):**
  Checkbox utama untuk mengaktifkan atau menonaktifkan seluruh fungsi proxy.
  Jika nonaktif, aplikasi otomatis menggunakan **Direct Connection** (tanpa proxy).

## ðŸ§  **Rekomendasi Teknis Tambahan**

1. Gunakan **Thread Pool Executor** atau **Async Task Manager** untuk manajemen multi-instance.
2. Terapkan **modular MVC/MVVM pattern** agar UI, logic, dan core engine terpisah dengan rapi.
3. Sediakan **Export Log (CSV/JSON)** dan **Auto Save Log**.
4. Tambahkan **Safe Mode** untuk menjalankan simulasi tanpa melakukan klik eksternal.
5. Gunakan **SQLite** untuk menyimpan data proxy, platform, dan konfigurasi.
6. Implementasikan **retry system** untuk flow gagal, dengan auto backoff.
7. Tambahkan **task scheduler** untuk menjalankan simulasi otomatis pada waktu tertentu.
